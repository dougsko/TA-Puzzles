    Fast Powers Test

    What is the fastest way to test if an int is a power of two?

    The trick to this question is to use a property found when using binary representation of numbers. If a number is a power of two, then in binary, it will be represented as a 1, followed by some number of zeros. If you subtract one from a power of two, then its binary representation will be all 1's. If you perform a bitwise AND operation of n and n-1, and n is a power of two, then the answer will always be zero. The one special edge case is if n itself is zero, in which case, we simply return false.

    The code below demonstrates this technique using java. Compile and run the application with the following commands:

    $ javac FastPowers.java
$ java FastPowers >number<

    Lot's of 0's

    How many zeros are on the end of n! ? Assume n is big. So n! is REALLY big.

    To solve this question, we will use a divide by 5^n trick. We continually divide n by powers of 5. At each division, we keep a running total of the integer value of the quotient. When the quotient is less than 1, we stop and the current cumulative sum equals the number of trailing zeros for any n!.

    What's Wrong With...

    ...this code?

    public static String parity(String msg)
{
	String parityStr = null;
	int n = msg.hashCode();
	switch(n % 2)
	{
	case 0:
		parityStr = "even";
		break;
	case 1:
		parityStr = "odd";
		break;
	}
	return parityStr;
}

    The problem with the code above is the use of the hashCode() method. The purpose of this method is to provide a unique integer value for any object. However, these integer values are meant to be dstributed evenly across a domain and have nothing to do with the even or oddness of the object.
